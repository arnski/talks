Einleitendes

Heute wollen wir euch etwas über Workflows erzählen. Wir werden es in weiten Teilen auf abstrakter Ebene abhandeln, sodass sich Teile auch auf andere SCM- Systeme übertragen lassen. Hier dargestellt: Wie Git- Workflows auf SVN- Veteranen wirken können. ;)

Auffrischung - Git

Auffrischung - Git

Hier wichtig: Anzahl Working Copies (Work Trees) gleich, jedoch mehr Repositories.. Das ergibt auch einen etwas veränderten Arbeitsfluß => next slide.

Auffrischung - Git

* Interaktion zw. lokalem Work Tree und lokalem Repo. * Interaktion zw. zwei Nicht-Servern möglich. * Interaktion zw. lokalem Repo und remote Repo.

Workflow - Centralized

  • Nur Master Branch
  • Arbeitsweise ähnlich wie von SVN gewohnt, aber nicht gleich - vgl. "Distributedness".
    • Git: Arbeit => Commit => Wdh. => Push
    • SVN: Arbeit => Commit
  • Lösen von Konflikten: Mit Hilfe von git add und git status

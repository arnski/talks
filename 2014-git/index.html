Einleitendes

Heute wollen wir euch etwas über Workflows erzählen. Wir werden es in weiten Teilen auf abstrakter Ebene abhandeln, sodass sich Teile auch auf andere SCM- Systeme übertragen lassen. Hier dargestellt: Wie Git- Workflows auf SVN- Veteranen wirken können. ;)

Auffrischung - Git

  • Hier Grafik mit Repo und Worktree / Server-Repo und Working Copy bei SVN

Workflow - Centralized

  • Nur Master Branch
  • Arbeitsweise ähnlich wie von SVN gewohnt
    • Arbeit
    • Commit
    • ...
    • Push
    • => Analog zu "Arbeit => Commit" bei SVN
  • Konfliktresolven: Mit Hilfe von git add und git status
